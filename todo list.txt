
=====================================================================
speed changes
=====================================================================
Some small customizations for speed are needed. 
Before starting to work, i suggest reading all the rest of this section because it is all related.

the following 3 customization changes would need to be added to make map making a lot better.
you can make it part of level data, but that would probly require an ingame interface.
diferent maps might need diferent speed, but ingame interface is proboly gona feel clunky. there is no right option here so pick what you want to do more.
if you do make it via map data, please also move ring size customization there (and remember old map data works diferent so it has to handle that)       
    1) cusomtization maximum speed bonus: to prevent long maps from being unplayably fast, a customization to set a maximum speed bonus value
    2) customization base speed: change the speed at which the player starts at, defaulting to 200
    3) cusomtization speed gain per cp: a customization config to set the amound of speed increase you receive per ring touched.

an example of ingame system would be: when u enter mode, up/down to select setting, left/rikght click to adjust setting, v to flip a value or reset to default.

=====================================================================
features general 
=====================================================================
- proximity invisibility: when everyone spawns in same location, people cant see anything because everyone is stacked in one room. proximity invisiblity makes players invisible to eachother if they are close to eachother.

- editor command: hold to leave editor + start new run > sometimes when you are editing, you want to try the course fom the start again. in retrospect, restart should be available, but it should close the editor before triggering the restart.

- editor speed calc: currently when in editor, your test mode keeps the last speed you had, wich can be completly different then what the user has on that cp. perhaps you can make it so it calculates the speed the user would have at that cp when you go in test mode, and if the user wants to test lower speed they can just use quick reset.

- boundry bypass orbs: place and size orbs that turn of your collision while you are inside them. this way u can fly trough a wall or boundry when ur inside it.

- control map navigation: make a way to get to other segments of control maps. perhaps you can use a command to teleport to different spawns or map objectives. more then 1 keypress so u dont acidentally go.

- 20+ person leaderboard: in gpk you can hold a button to toggle a full screen hud leaderboard with abou 20 entries.

- death loop protection: both editor and game. if you place a cp, or a kill zone changes, you can continuesly spawn in death zone and crash loop

- edit outlines: show person who is editing when they are invis with outlines so u know whats going on when you are editing with friends.

- editor: have a button to set a cp as the test cp. on using quick reset you would appear at that cp instead of the normal cp. this way you can test run a group of cps in order

- lion has a way to bypass some death boundries. see feedback channel on discord.

- time remaining: the remaining lobby time should be displayed. consider how to handle infinite.

- credit hint: the default cusomtization value for credit should hint that you can set this value in the configuration rule so people can find it.
